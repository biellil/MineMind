**Adicionar a função makeParkPromise** (FORA de startCognitiveLoop, após o `const sleep = ...` — ou remover o sleep e adicionar makeParkPromise no lugar):

```typescript
type WakeReason = 'actionFinished' | 'timeout' | 'abort'

/**
 * Estaciona o driver até um dos três eventos: actionFinished (próximo passo pronto),
 * sinal de abort (bot.end/desconexão), ou timeout-piso de segurança.
 * D-01/D-09: o timeout-piso é REDE DE SEGURANÇA, não o motor do loop.
 * D-11: cleanup obrigatório remove listeners e clearTimeout ao acordar.
 */
function makeParkPromise(
  triggerBus: TriggerBus,
  abortSignal: AbortSignal,
  nextWakeMs: number,
): Promise<WakeReason> {
  return new Promise<WakeReason>((resolve) => {
    let timer: ReturnType<typeof setTimeout> | undefined

    const onActionFinished = () => { cleanup(); resolve('actionFinished') }
    const onAbort = () => { cleanup(); resolve('abort') }

    timer = setTimeout(() => { cleanup(); resolve('timeout') }, nextWakeMs)
    triggerBus.once('actionFinished', onActionFinished)
    abortSignal.addEventListener('abort', onAbort, { once: true })

    function cleanup() {
      clearTimeout(timer)
      triggerBus.off('actionFinished', onActionFinished)
      // NÃO remover o abortSignal listener (once: true é suficiente)
    }
  })
}
```

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| TriggerBus events → makeParkPromise | Eventos do TriggerBus resolvem a Promise do race — um emit falso aceleraria o loop |
| sessionAbort signal → bot.end | O AbortController.abort() deve ser chamado exatamente uma vez no end; chamadas extras são no-op (idempotente) |
| result.enteredIdle → nextWakeMs | Campo do retorno do grafo determina por quanto tempo o bot fica em park |

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-07.1-08 | Denial of Service | makeParkPromise / timeout-piso muito baixo | mitigate | nextWakeMs padrão = navigateTimeoutMs (30s) quando não idle; idleWakeIntervalMs (60s) quando idle genuíno — D-09 nunca usar minTickMs como timeout-piso |

| T-07.1-11 | Denial of Service | deadlock no park se bot.end não emitir | mitigate | sessionAbort.abort() via bot.once('end') resolve o race imediatamente; timeout-piso como última salvaguarda |
