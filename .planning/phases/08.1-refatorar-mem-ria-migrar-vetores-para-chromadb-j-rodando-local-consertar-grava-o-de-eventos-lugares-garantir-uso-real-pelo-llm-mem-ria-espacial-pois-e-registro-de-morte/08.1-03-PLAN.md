    1. **Estado do breaker** (vive no closure, por instância):

    2. **`warnOfflineDebounced(b, now)`** (D-22): emite `console.warn("[chroma] OFFLINE — memória vetorial desativada. Inicie 'chroma run' (localhost:8000) para habilitar.")` no máximo a cada `config.chromaWarnDebounceMs`, atualizando `b.lastWarnAt`.

    3. **`withBreaker<T>(b, fn, fallback)`** (Pattern 3): closed→open após `config.chromaFailThreshold` falhas; open por `config.chromaCooldownMs` (durante o cooldown retorna `fallback` + warn debounced, sem chamar `fn`); após o cooldown vira half-open e tenta `fn`; sucesso → closed/failures=0; falha → incrementa failures, abre se cruzar o threshold, retorna `fallback`. NUNCA propaga exceção.

    4. **Timeout de fetch** (Pitfall 5): cada chamada ao Chroma deve cortar em `config.chromaFetchTimeoutMs`. Como o repasse de `signal` pelo cliente é INCERTO (Open Question 1 da pesquisa), usar `Promise.race([fn(), timeout])` onde `timeout` rejeita após `chromaFetchTimeoutMs` (helper `withTimeout<T>(p, ms)`). Isso garante o corte independentemente do cliente repassar `signal`. (Opcionalmente também passar `fetchOptions` no construtor, mas o `Promise.race` é a garantia.)

    5. **`ensureCollection()`** (D-23/D-03/D-05): cria o `ChromaClient({ host: config.chromaHost, port: config.chromaPort, ssl: config.chromaSsl })` e chama `getOrCreateCollection({ name: config.chromaCollection, embeddingFunction: null, configuration: { hnsw: { space: 'cosine' } } })`. Memoizar a collection (lazy, 1x). Envolto em withBreaker (fallback null). Se retornar null, os métodos add/query viram no-op/[].

    6. **`health()`** (Pattern 4/D-22): `await withTimeout(client.heartbeat(), timeout)`; retorna boolean. Em falha, warn debounced e retorna false. Usado no boot pelo Plan 04.

    7. **`addVector({ id, embedding, metadata, document })`** (Pattern 2): valida `embedding.length === EMBEDDING_DIM` (Pitfall 2 — se divergir, loga e no-op); `ids: [String(id)]`; coage metadata para `string|number|boolean` (use `player ?? ''` no caller — aqui só repassa). Envolto em withBreaker (fallback: void). NÃO embeda nada — recebe o vetor pronto.

    8. **`queryVectors({ queryEmbedding, nResults, player })`** (Pattern 2): chama `collection.query({ queryEmbeddings: [queryEmbedding], nResults, where: player ? { player: { $eq: player } } : undefined })`; desempacota o retorno DOUBLE-NESTED (`res.ids[0]`, `res.distances[0]`) e retorna `Array<{ id: string; distance: number }>`. Envolto em withBreaker (fallback: `[]`). Validar `queryEmbedding.length === EMBEDDING_DIM` antes (senão retorna []).
