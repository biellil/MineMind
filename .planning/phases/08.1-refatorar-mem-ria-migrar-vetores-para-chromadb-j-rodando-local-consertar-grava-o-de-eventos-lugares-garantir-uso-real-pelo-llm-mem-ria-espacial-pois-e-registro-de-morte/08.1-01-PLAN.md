       ```ts
       if (version < 2) {
         // D-14: coluna type nullable (ALTER ADD COLUMN no SQLite NÃO pode ser NOT NULL sem default).
         // Guardar contra "duplicate column" se a migração já rodou parcialmente (ver passo 3).
         addColumnIfMissing(db, 'places', 'type', 'TEXT')
         // D-19: tabela de lições duráveis (conhecimento generalizado, distinto de events).
         db.run(`CREATE TABLE IF NOT EXISTS lessons (
           id INTEGER PRIMARY KEY,
           text TEXT NOT NULL,
           confidence REAL NOT NULL DEFAULT 0.5,
           reinforce_count INTEGER NOT NULL DEFAULT 0,
           contradict_count INTEGER NOT NULL DEFAULT 0,
           last_seen INTEGER NOT NULL,
           created_at INTEGER NOT NULL )`)

       }
       db.run(`PRAGMA user_version = ${SCHEMA_VERSION}`)
       ```

    3. Criar um helper `addColumnIfMissing(db, table, column, typeDdl)` que torna o `ALTER TABLE ADD COLUMN` idempotente (SQLite não tem `ADD COLUMN IF NOT EXISTS`). Implementar lendo `PRAGMA table_info(<table>)` e só executando o ALTER se a coluna não existir:
       ```ts
       function addColumnIfMissing(db: Database, table: string, column: string, typeDdl: string): void {
         const cols = db.prepare(`PRAGMA table_info(${table})`).all() as Array<{ name: string }>
         if (cols.some((c) => c.name === column)) return
         db.run(`ALTER TABLE ${table} ADD COLUMN ${column} ${typeDdl}`)
       }
       ```
       NOTA: `PRAGMA table_info` não aceita placeholder `?` para o nome da tabela — interpolar direto (os valores são literais do código, não input externo).

    4. Adicionar e exportar os tipos TS dos novos artefatos (consumidos pelos Plans 05/06):

       ```ts
       export type PlaceType = 'base' | 'resource' | 'danger' | 'village' | 'landmark'

         contradict_count: number
         last_seen: number
         created_at: number
       }
       ```
